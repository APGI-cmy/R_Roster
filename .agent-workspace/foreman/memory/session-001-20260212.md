### Decisions Made

**Decision 1: Deterministic PR Classification**
- **What**: Implemented rule-based PR classification (not inference-based)
- **Why**: Eliminates misapplication errors; follows canonical pattern from MERGE_GATE_INTERFACE_STANDARD.md
- **Impact**: Predictable gate behavior, clear classification logic

**Decision 2: Evidence-First Validation**
- **What**: Check for evidence artifacts before running validation scripts
- **Why**: Aligns with MERGE_GATE_PHILOSOPHY.md confirmatory pattern; avoids redundant work
- **Impact**: Faster gate execution, clear error messages

**Decision 3: Legacy Gates Coexistence**
- **What**: Keep existing gates functional but make optional
- **Why**: Grace period allows smooth migration; reduces risk of blocking valid work
- **Impact**: Three-phase deprecation timeline (now through 2026-04-12)

**Decision 4: Foreman Memory Management**
- **What**: 5-session rotation with archive, personal learning never rotates
- **Why**: Per FOREMAN_MEMORY_PROTOCOL.md requirements
- **Impact**: Persistent learning while managing workspace size

**Decision 5: Machine-Readable Gate Results**
- **What**: JSON schema for gate results with required fields
- **Why**: Per EVIDENCE_ARTIFACT_BUNDLE_STANDARD.md; enables programmatic validation
- **Impact**: Eliminates narrative-only claims, enables fail-fast validation
